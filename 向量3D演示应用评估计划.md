## 一、需求理解与范围界定

**核心目标**：在三维笛卡尔坐标系中，实时展示一个或多个空间向量，并直观呈现：
- 向量在坐标轴上的分向量（分量箭头）
- 分解为两个或三个方向的平行四边形/平行六面体法则
- 向量与坐标轴夹角、方向余弦等三角函数关系
- 所有元素均可交互控制

**用户交互**：
- 视角控制（拖动旋转、预设固定视角）
- 缩放（滚轮、UI按钮）
- 向量的增删与参数调节（长度、方向、粗细、颜色）
- 分量/分解的可视化开关
- 背景色自定义

---

## 二、技术方案评估

### 2.1 实现途径选择

| 方案 | 优点 | 缺点 | 适合度 |
|------|------|------|--------|
| **Web 3D（Three.js）** | 无安装、跨平台、丰富3D生态、易集成UI、实时渲染 | 对浏览器有一定性能要求 | ★★★★★ |
| 桌面应用（Electron + Three.js） | 可离线、系统API访问 | 打包体积大、开发成本高 | ★★★ |
| Unity WebGL | 画面精美、物理引擎成熟 | 加载慢、Web集成复杂、包体大 | ★★★ |
| Python（VTK/Mayavi） | 科学计算强 | 交互式Web部署困难，非实时交互性弱 | ★★ |

**推荐方案：纯Web前端应用，基于Three.js实现3D渲染，配合原生HTML/CSS/JS构建UI面板。**  
- 最简单部署：静态网页即可运行，可托管于GitHub Pages等。
- 开发灵活：Three.js社区成熟，示例丰富，可快速实现坐标轴、箭头、标签、光线等。
- UI控制可以完全独立，无需受引擎限制。

### 2.2 技术栈建议

- **3D引擎**：Three.js（r160+），使用其`Arrow`、`Line`、`TextGeometry`/CSS2DRenderer精灵文字等
- **数学库**：Three.js内置`Vector3`，也可配合`gl-matrix`做额外运算
- **UI框架**：纯原生或轻量级（Vue/React可选），推荐用Vue 3 + Naive UI 或直接原生JS以降低依赖
- **构建工具**：Vite（快速开发及打包）
- **其他辅助**：
  - `OrbitControls`（Three.js官方提供，实现旋转/平移/缩放）
  - `TransformControls` 可选（拖拽向量端点调整方向/长度）
  - `CSS2DRenderer`（用于清晰显示向量标签、角度标注）

## 三、功能详细分析与设计要点

### 3.1 场景元素结构

```
Scene
├── Grid & Floor (可选的参考网格)
├── Axes3D
│   ├── X轴线（带箭头、标签）
│   ├── Y轴线
│   ├── Z轴线
│   └── 刻度标记
├── VectorGroup i (每个向量一组)
│   ├── MainArrow (主体向量箭头)
│   ├── ComponentArrows
│   │   ├── CompX (X分量，虚线/半透明可开关)
│   │   ├── CompY
│   │   └── CompZ
│   ├── DecompositionLines (平行四边形/六面体辅助虚线)
│   └── AngleArcs (与各轴夹角弧线 + 标签)
└── Labels (向量的名称、数值、角度，使用CSS2D或sprite)
```

### 3.2 关键实现细节

- **向量分解可视化**：当开启“分解”时，从向量终点分别向坐标平面/轴引垂直虚线，构成投影矩形（2D）或长方体（3D），并在轴上显示分量箭头。
- **三角函数关系**：显示向量与三个轴正方向的夹角α、β、γ，以及对应的cos值标签，还可显示 `cos²α+cos²β+cos²γ=1` 的验证信息（可选）。
- **向量参数调节**：
  - 方向：可通过直接输入角度（与轴夹角或球坐标θ, φ）、或输入终点坐标、或拖拽箭头终点（TransformControls）来改变。
  - 长度：滑块调节模长（保持方向不变），或直接输入数值。
  - 粗细：修改箭头圆柱半径。
- **预设视角**：提供按钮快速跳转到XOY平面、XOZ平面、YOZ平面视角，以及等轴测视角。

### 3.3 控制UI布局

采取侧边栏或悬浮面板，分组管理：
- **全局设置**：背景颜色、网格显示/隐藏、坐标轴颜色
- **向量列表**：每条向量一行，带删除按钮，“添加向量”按钮
- **选中向量参数**：长度、方向角（θ, φ或输入x,y,z分量）、颜色、粗细、是否显示分量、是否显示分解虚线、是否显示角度弧
- **视角预设按钮**：XOY(top), XOZ(front), YOZ(right), isometric

### 3.4 性能考量
- 向量数量不会很大（一般<10），无需LOD或大量优化。
- 注意CSS标签数量，可用Canvas纹理做标签以减轻DOM负担，但CSS2D更清晰灵活，几个标签完全没问题。

## 四、初步开发计划（迭代步骤）

### 第一阶段：基础框架搭建（1-2天）
1. 初始化Vite项目，安装Three.js。
2. 搭建基本场景：透视相机、WebGL渲染器、OrbitControls、光照（环境光+方向光）。
3. 绘制3D坐标轴（用Arrow或Cylinder+Cone），添加轴标签（CSS2D）。
4. 添加一个参考网格（可选，用GridHelper）。
5. 实现背景色修改功能（更新`renderer.setClearColor`或场景背景）。

### 第二阶段：核心向量系统（2-3天）
1. 实现向量数据模型（向量类）：包含起点（默认原点）、终点、颜色、粗细等属性。
2. 绘制主向量（`Arrow`），根据属性更新。
3. 绘制分量箭头：将向量终点投影到三个坐标平面，再分别投影到轴上，生成X、Y、Z方向的分量箭头（长度对应分量值，颜色与轴匹配）。
4. 绘制分解辅助线（虚线）：从向量终点向各分量箭头的末端连线。
5. 实现向量的添加与删除（UI列表联动）。

### 第三阶段：交互与参数控制（2天）
1. 实现向量参数面板：长度滑块、方向角输入（球坐标转笛卡尔坐标）、或直接输入x,y,z分量。
2. 粗细与颜色调节。
3. 分量箭头的显示/隐藏开关。
4. 分解虚线的显示/隐藏。
5. 向量名称可编辑。

### 第四阶段：三角函数关系展示（1-2天）
1. 计算向量与各轴夹角：使用`Math.acos(component/length)`。
2. 绘制角度弧线：借助THREE.EllipseCurve或自定义BufferGeometry绘制从轴到向量的扇环。
3. 添加角度值和cos值的标签，更新时同步变动。
4. 可选展示恒等式验证文本。

### 第五阶段：视角预设与UI细节打磨（1天）
1. 实现预设视角按钮，使用GSAP或简单动画过渡`camera.position`和`controls.target`。
2. 缩放UI按钮（步进缩放）。
3. 响应式布局调整，移动端基本兼容（至少保证UI可用）。

### 第六阶段：测试与部署（0.5-1天）
1. 跨浏览器测试（Chrome, Firefox, Edge, Safari）。
2. 打包优化，部署到GitHub Pages或Netlify。

## 五、潜在风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Three.js箭头连接处不够美观 | 视觉效果 | 可自己构建圆柱+圆锥组合体，完全控制材质与比例 |
| CSS2D标签在复杂视角下遮挡或重叠 | 可读性 | 添加基于深度的透明度或优先显示选中向量的标签；允许拖拽标签偏移 |
| 角度弧绘制复杂 | 开发周期 | 初期可用简单的线段和纹理精灵表示；后续优化成弧线 |
| 移动端触屏交互不友好 | 用户体验 | 确认OrbitControls支持触摸；UI按钮需足够大；简化移动版UI |
