# AGENTS.md

## 项目目标

CartesianVectors 是一个中文优先的 3D 笛卡尔坐标系向量教学演示应用。第一版应帮助学习者直观看到空间向量、XYZ 分量、分解辅助线、夹角关系和方向余弦。

## 开发原则

- 优先保证教学表达清晰、交互稳定、代码可维护。
- 首屏必须是可交互应用，不创建营销首页。
- 保持轻量前端实现，第一版不引入后端、登录、数据库或云端保存。
- 不做与当前任务无关的重构或格式 churn。
- 修改功能或代码前必须评估改动边界，保持变更范围最小。
- 不要修改已实现且无问题的功能、无关文件、系统设置或外部配置；如确有必要，必须先向用户确认。

## 技术约束

- 使用 Vite、TypeScript、Three.js。
- 使用 Three.js `OrbitControls` 和 `CSS2DRenderer`。
- 完整 3D 视图依赖浏览器 WebGL；必须保留 WebGL 不可用时的 2D fallback 或清晰提示。
- MVP 阶段不引入 React、Vue 或大型 UI 框架。
- 前端依赖由 npm 管理；本地调试和演示推荐优先通过 Conda 环境提供 Node/npm。
- 不建议直接依赖系统 Node/npm；仅在排障或环境创建失败时临时使用。

## UI 与视觉规则

- 第一版 UI 语言使用中文，数学符号可保留 `x/y/z`、`|v|`、`α`、`β`、`γ`、`cos(...)`。
- 坐标轴颜色保持稳定：X 红、Y 绿、Z 蓝。
- 坐标轴轴身使用虚线或点划线，主向量保持实体箭头。
- 分量箭头跟随对应向量颜色，但应使用透明度或浅色调与主向量区分。
- 标签和控制面板不能遮挡主要操作；中文文本不能溢出按钮、输入框或面板。
- 使用 `CSS2DRenderer` 时，重绘或删除对象前必须清理旧标签 DOM，避免页面残留死标签。
- 视觉风格应偏教学工具和实验台，不做装饰性 landing page。

## 数学约束

- 向量主状态始终使用笛卡尔分量 `x/y/z`。
- 方向模式的 `|v|`、Yaw、Pitch 必须实时换算回 `x/y/z`，不要作为长期主状态。
- 长度、单位向量、方向余弦、夹角等均由 `x/y/z` 派生计算。
- 零向量允许存在，但方向、夹角和方向余弦必须显示为不可定义，不能出现 `NaN`。
- 默认向量起点为原点；任意起点向量属于后续扩展。

## 测试要求

- 提交前至少运行 `npm run build`。
- 推荐使用 `conda run -n cartesian-vectors npm run build` 完成构建验证。
- 人工验证正分量、负分量、混合分量、零向量。
- 人工验证添加、删除、隐藏向量不会影响其他向量。
- 人工验证反复开关“角度关系”“分量显示”不会残留标签。
- 视觉检查桌面布局和基本移动端布局。

## 禁止事项

- 不要把中文默认 UI 改成英文。
- 不要引入后端服务作为第一版依赖。
- 不要把派生数学量作为长期主状态存储。
- 不要使用破坏性 Git 命令回滚他人改动。
- 不要要求开发者必须污染全局 Node/npm 环境。
